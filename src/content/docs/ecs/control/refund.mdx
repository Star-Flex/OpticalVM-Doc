当您不再需要使用 ECS 实例时，可以申请退款。退款金额根据实例的剩余使用时长按比例计算，确保费用公平透明。

退款流程包括：

- **申请退款**：提交退款申请后，实例将被删除并释放资源
- **费用计算**：系统自动计算所有订单的剩余价值总和
- **资金返还**：退款金额将返还至您的账户余额

:::caution[重要提示]

- 退款操作将**立即删除实例**，请务必提前备份数据
- 已使用天数**向上取整**（使用不足 1 天按 1 天计算）
- 已过期的订单或实例无法退款
- 账户余额可以在财务中心进行提现（原路退还）
  :::

## 退款提现流程

<Steps>

1. **工单发起退款**

   在工单系统中提交退款申请，管理员将在 1-3 个工作日内处理。

2. **审核通过后费用到账**

   审核通过后，退款金额将自动返还至您的账户余额（钱包）。

3. **账户余额发起提现**

   前往财务中心 - 账户钱包，发起提现申请，退款金额将原路退还至您的支付账户。

</Steps>

:::note[提现说明]

- 提现金额将**原路退还**至原支付方式
- 提现到账时间：1-3 个工作日（具体以支付渠道为准）
- 账户余额也可用于购买其他服务，无需提现

:::

## 退款费用计算规则

退款采用**按天计费**的方式，仅退还实例剩余时长对应的费用。

**核心公式：**

```txt
退款金额 = Σ(每个订单的剩余价值)
单个订单剩余价值 = (剩余天数 ÷ 总天数) × 订单实付金额
```

**详细计算步骤：**

1. **计算订单总天数**

   ```
   总天数 = ⌈(订单到期时间 - 订单开始时间) ÷ 24小时⌉
   ```

   注：⌈ ⌉ 表示向上取整

2. **计算已使用天数**

   ```
   已使用天数 = ⌈(当前时间 - 订单开始时间) ÷ 24小时⌉
   ```

   注：**使用不足 1 天按 1 天计算**

3. **计算剩余天数**

   ```
   剩余天数 = 总天数 - 已使用天数
   ```

4. **计算单个订单退款金额**

   ```
   订单退款金额 = (剩余天数 ÷ 总天数) × 订单实付金额
   ```

5. **计算总退款金额**
   ```
   总退款金额 = 新购订单退款 + 续费订单退款 + 升级订单退款
   ```

## 计算示例

### 示例 1：单个新购订单退款

**场景：**

- **购买时间**：2025-01-01 10:00
- **购买时长**：30 天
- **实付金额**：¥300
- **当前时间**：2025-01-11 14:30（使用了 10 天 4 小时 30 分钟）

**计算过程：**

```
步骤 1：计算订单总天数
  到期时间 = 2025-01-01 10:00 + 30天 = 2025-01-31 10:00
  总天数 = 30 天

步骤 2：计算已使用天数（向上取整）
  实际使用时长 = 10 天 4 小时 30 分钟
  已使用天数 = ⌈10.1875⌉ = 11 天

步骤 3：计算剩余天数
  剩余天数 = 30 - 11 = 19 天

步骤 4：计算退款金额
  退款金额 = (19 ÷ 30) × ¥300 = ¥190.00
```

**最终退款：¥190.00**

### 示例 2：多个订单退款（新购 + 续费）

**场景：**

- **新购订单**
  - 购买时间：2025-01-01 10:00
  - 购买时长：30 天
  - 实付金额：¥300
- **续费订单**
  - 续费时间：2025-01-20 10:00（未过期时续费）
  - 续费时长：30 天
  - 实付金额：¥300
- **当前时间**：2025-02-05 14:00

**计算过程：**

```
订单 1（新购）：
  订单开始时间：2025-01-01 10:00
  订单到期时间：2025-01-31 10:00
  总天数：30 天
  已使用天数：⌈35.17⌉ = 36 天（已超过订单时长）
  剩余天数：0 天（订单已过期）
  订单退款：¥0

订单 2（续费）：
  订单开始时间：2025-01-31 10:00（从上个订单到期时间开始）
  订单到期时间：2025-03-02 10:00
  总天数：30 天
  已使用天数：⌈5.17⌉ = 6 天
  剩余天数：30 - 6 = 24 天
  订单退款：(24 ÷ 30) × ¥300 = ¥240.00

总退款金额：¥0 + ¥240.00 = ¥240.00
```

**最终退款：¥240.00**

### 示例 3：包含升级订单的退款

**场景：**

- **新购订单**
  - 购买时间：2025-01-01 10:00
  - 购买时长：30 天
  - 实付金额：¥300
- **升级订单**
  - 升级时间：2025-01-10 10:00
  - 升级费用：¥100（剩余 21 天的配置增量费用）
  - 到期时间：2025-01-31 10:00（与原订单相同）
- **当前时间**：2025-01-15 10:00

**计算过程：**

```
订单 1（新购）：
  总天数：30 天
  已使用天数：⌈14⌉ = 14 天
  剩余天数：30 - 14 = 16 天
  订单退款：(16 ÷ 30) × ¥300 = ¥160.00

订单 2（升级）：
  订单开始时间：2025-01-10 10:00
  订单到期时间：2025-01-31 10:00
  总天数：21 天
  已使用天数：⌈5⌉ = 5 天
  剩余天数：21 - 5 = 16 天
  订单退款：(16 ÷ 21) × ¥100 = ¥76.19

总退款金额：¥160.00 + ¥76.19 = ¥236.19
```

**最终退款：¥236.19**

## 特殊情况说明

### 过期后续费再退款

如果实例过期后才续费，续费订单的开始时间从**续费时间**开始计算，而不是从过期时间开始。

**示例：**

```
新购订单：2025-01-01 购买 30 天，到期时间 2025-01-31
过期后续费：2025-02-10 续费 30 天
  - 续费订单开始时间：2025-02-10（续费时间）
  - 续费订单到期时间：2025-03-12
  - 不是从 2025-01-31 开始计算
```

### 已使用时间不足 1 天

根据**向上取整**规则，即使只使用了几分钟，也按 1 天计算。

**示例：**

```
购买时间：2025-01-01 10:00
当前时间：2025-01-01 10:30（仅使用 30 分钟）
已使用天数：⌈0.02⌉ = 1 天
```

:::tip[温馨提示]

- 退款金额将返还至您的账户余额，可用于购买其他服务
- 退款后，与实例相关的所有订单将标记为"已退款"状态

:::

## 发起退款

ECS 实例退款需要通过**工单系统**申请处理。请按照以下步骤提交退款工单。

:::caution[数据安全提醒]

退款操作将**永久删除实例及其所有数据**，包括：

- 系统盘和数据盘中的所有文件
- 实例配置和快照
- 网络配置和端口映射

**退款前请务必备份所有重要数据，操作无法撤销！**

:::

### 工单模板

提交退款工单时，请使用以下模板格式：

**标题：**

```
ecs-xxx 需要申请退款
```

将 `xxx` 替换为您的实例 ID，例如：`ecs-abc123 需要申请退款`

**内容：**

```
ecs-xxx 申请退款，数据我已保留可以直接执行退款操作
```

将 `xxx` 替换为您的实例 ID，例如：`ecs-abc123 申请退款，数据我已保留可以直接执行退款操作`
